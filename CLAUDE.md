# FoodMax AI 规则

## 1. 写代码先写测试
所有代码变更从失败测试开始。bug 复现测试先于修复；新功能行为测试先于实现。

## 2. 改动最小化
匹配已有风格；不顺手改无关代码；发现不相关问题口头汇报，不动手。

## 3. 使用团队 skill
- 写 PR 描述：调用 `/foodmax-pr-description`
- 创建新模块脚手架：调用 `/foodmax-new-module`

## 4. 不引入未审计依赖
任何新 npm/python 依赖在 PR 描述里单独标注，附原因 + 替代方案评估。

## 5. 失败响应
遇到失败先复现 → 再定位 → 再修复。不要"试一下"猜性修复。
